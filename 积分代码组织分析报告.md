# PEEC/MoM 积分代码组织分析报告

## 当前代码组织状况

### 1. 公共积分函数库 (`src/core/electromagnetic_kernels.h/c`)

**已实现的公共积分函数：**
- `integrate_triangle_singular()` - 三角面积分（带奇异性处理）
- `integrate_rectangle_regular()` - 矩形面积分
- `integrate_wire_thin()` - 线积分（薄线近似）
- `green_function_free_space()` - 自由空间Green函数
- `green_function_layered_media()` - 多层介质Green函数

**设计目标：**
- 为MoM和PEEC提供统一的积分接口
- 处理奇异性问题
- 支持多层介质

### 2. MoM中的积分使用情况

**位置：** `src/core/core_assembler.c`

**现状：**
- 定义了 `integrate_triangle_triangle()`, `integrate_rectangle_rectangle()`, `integrate_wire_wire()` 等函数
- **但这些函数目前是空实现（占位符）**
- 代码注释中提到要使用 `electromagnetic_kernels` 库，但实际未调用
- 有转换代码将几何结构转换为 `triangle_element_t`，但未实际使用

**问题：**
```c
// core_assembler.c 第300-316行
static complex_t kernel_self_term_triangle(...) {
    triangle_element_t tri_elem;
    // 转换几何结构...
    // Use the electromagnetic_kernels integration function
    (void)formulation; (void)k; return (complex_t){0.0, 0.0};  // ❌ 空实现
}
```

### 3. PEEC中的积分使用情况

**位置：** `src/solvers/peec/peec_integration.c` (新创建)

**现状：**
- **完全独立的积分实现**，未使用公共积分函数
- 自己实现了高斯积分点/权重
- 自己实现了Neumann公式积分
- 自己实现了双重面积分

**问题：**
- 代码重复：高斯积分在多个地方实现
- 未复用公共的奇异性处理
- 未使用公共的Green函数

### 4. 代码组织问题总结

#### ❌ 问题1：MoM未真正使用公共积分函数
- `core_assembler.c`中的积分函数是占位符
- 应该调用 `electromagnetic_kernels` 中的函数

#### ❌ 问题2：PEEC未使用公共积分函数
- `peec_integration.c`完全独立实现
- 应该复用 `electromagnetic_kernels` 中的函数

#### ❌ 问题3：公共积分函数功能不完整
- `integrate_triangle_singular` 等函数实现过于简化
- 缺少双重积分（PEEC需要）
- 缺少Neumann公式的完整实现

#### ❌ 问题4：代码重复
- 高斯积分在多个地方实现
- Green函数计算可能重复

## 结论

**当前代码组织不符合"公用积分代码"的设计原则。**

**主要问题：**
1. MoM和PEEC都没有真正使用公共积分函数
2. PEEC新加的积分代码完全独立，未复用公共代码
3. 公共积分函数功能不完整，无法满足PEEC需求

**建议：**
1. **立即重构**：统一使用公共积分函数库
2. **增强功能**：完善公共积分函数，支持PEEC的双重积分需求
3. **代码复用**：移除重复实现，统一使用公共函数
4. **测试验证**：确保重构后功能正确
