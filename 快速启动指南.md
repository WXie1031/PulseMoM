## 🚀 快速开始

### 步骤1：打开Visual Studio项目

**方法1：直接打开**
1. 打开Visual Studio 2022
2. 文件 → 打开 → 项目/解决方案
3. 导航到：`D:\Project\MoM\PulseMoM\build\PEEC_MoM_Unified_Framework.sln`
4. 点击"打开"

**方法2：双击打开**
- 直接双击 `build\PEEC_MoM_Unified_Framework.sln` 文件

**方法3：使用批处理脚本**
- 运行 `build_vs.bat`，脚本会自动打开Visual Studio

## 🔨 编译项目

### 选择配置

在Visual Studio顶部工具栏：
- **解决方案配置**：选择 `Debug` 或 `Release`
- **解决方案平台**：选择 `x64`

### 编译

1. **编译整个解决方案**：
   - 菜单：生成 → 生成解决方案 (Build → Build Solution)
   - 快捷键：`Ctrl + Shift + B`

2. **编译单个项目**：
   - 在解决方案资源管理器中，右键点击项目
   - 选择"生成" (Build)

### 编译输出

编译完成后，查看输出窗口：
- 成功：显示 "生成成功"
- 失败：查看错误信息，修复后重新编译

## 🐛 调试配置

### 设置启动项目

1. 在解决方案资源管理器中，找到可执行文件项目：
   - `mom_cli` - MoM命令行工具
   - `peec_cli` - PEEC命令行工具
   - 或其他测试项目

2. 右键点击项目 → 设为启动项目 (Set as StartUp Project)

### 配置调试参数

1. 右键点击启动项目 → 属性 (Properties)
2. 配置属性 → 调试 (Debugging)
3. 设置以下参数：

**命令参数示例（MoM）**：
```
--geometry examples/test.stl --mesh examples/test.msh --frequency 1e9 --output results/output
```

**命令参数示例（PEEC）**：
```
--input examples/pcb.gds --output results/peec_output --freq 1:10:50
```

**工作目录**：
```
$(ProjectDir)..\..
```

### 设置断点

**关键调试位置**：

1. **MoM求解器**：
   - `src/solvers/mom/mom_solver_unified.c`
     - 第462行：`assemble_impedance_matrix_basic()` - 阻抗矩阵组装
     - 第428行：`solve_linear_system_basic()` - 线性系统求解
     - 第280行：`compute_problem_characteristics()` - 问题特征计算

2. **PEEC求解器**：
   - `src/solvers/peec/peec_solver_unified.c`
     - 部分元件提取函数
   - `src/solvers/peec/peec_integration.c`
     - 积分计算函数

3. **积分函数**：
   - `src/core/core_assembler.c`
     - 第226行：`integrate_triangle_triangle()` - 三角积分
   - `src/core/electromagnetic_kernels.c`
     - 各种积分函数

### 开始调试

1. **设置断点**：在代码行号左侧点击，出现红色圆点
2. **开始调试**：
   - 菜单：调试 → 开始调试 (Debug → Start Debugging)
   - 快捷键：`F5`
   - 工具栏：点击"本地Windows调试器"按钮

3. **调试控制**：
   - `F5` - 继续执行
   - `F10` - 逐过程 (Step Over)
   - `F11` - 逐语句 (Step Into)
   - `Shift+F11` - 跳出 (Step Out)
   - `Shift+F5` - 停止调试

### 查看变量

**调试窗口**：
- **局部变量**：调试 → 窗口 → 局部变量 (Locals)
- **监视**：调试 → 窗口 → 监视 → 监视1 (Watch)
- **调用堆栈**：调试 → 窗口 → 调用堆栈 (Call Stack)
- **内存**：调试 → 窗口 → 内存 → 内存1 (Memory)

**快速监视**：
- 选中变量 → 按 `Shift+F9`
- 或右键 → 快速监视 (QuickWatch)

## 🔧 常见问题

### 问题1：找不到头文件

**解决**：
1. 检查项目属性 → C/C++ → 常规 → 附加包含目录
2. 确保包含 `src` 目录

### 问题2：链接错误

**解决**：
1. 检查项目属性 → 链接器 → 常规 → 附加库目录
2. 检查项目属性 → 链接器 → 输入 → 附加依赖项
3. 确保所有必需的库都已链接

### 问题3：运行时错误

**解决**：
1. 启用异常断点：调试 → 窗口 → 异常设置
2. 勾选所有C++异常
3. 重新运行，查看异常信息

### 问题4：程序立即退出

**解决**：
1. 在 `main()` 函数末尾设置断点
2. 或在 `main()` 函数末尾添加：
   ```c
   printf("Press Enter to exit...\n");
   getchar();
   ```

## 🎯 调试技巧

### 技巧1：条件断点

1. 设置断点
2. 右键点击断点 → 条件
3. 输入条件，例如：`i > 100`

### 技巧2：数据断点

1. 调试 → 新建断点 → 数据断点
2. 输入变量名或地址
3. 当变量值改变时停止

### 技巧3：日志点

1. 右键点击断点 → 操作
2. 勾选"记录消息"
3. 输入日志消息，例如：`变量i的值: {i}`

### 技巧4：调用堆栈导航

1. 在调用堆栈窗口中
2. 双击任意函数
3. 跳转到该函数的代码位置
