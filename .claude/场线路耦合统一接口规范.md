# FIELD–CIRCUIT–LINE UNIFIED COUPLING INTERFACE

## 跨 FDTD–MoM 的统一场–路–线耦合接口规范（设计稿）

> **适用范围**
>
> * FDTD（时域、Yee 网格、GPU）
> * MoM / PEEC（频域或时域、积分算子）
> * MTL（传输线、多导体、端口网络）
> * 混合仿真（FDTD–MoM–MTL–Circuit）
>
> **设计目标**
> 在**不破坏各方法内部数值结构**的前提下，建立一个**方法无关、物理一致、可扩展**的场–路–线耦合接口。

---

## 1. 设计总原则（必须同时满足）

### 原则 P1：接口只表达“物理交换量”，不表达数值细节

* 不暴露网格、基函数、矩阵、时间步
* 只暴露：电压、电流、场量、功率、阻抗等物理量

### 原则 P2：FDTD / MoM / MTL 地位完全对等

* 没有“主方法”
* 所有方法都通过 **Coupling Interface** 交换信息

### 原则 P3：耦合是显式建模，不是“偷偷加项”

* 每一个耦合项必须：

  * 有明确物理意义
  * 有可追溯能量路径
  * 可独立验证

## 3. 统一耦合对象（核心抽象）

### 3.1 Coupling Entity（耦合实体）

所有耦合都围绕以下三类实体展开：

```text
CouplingEntity :=
  FieldEntity    (连续场)
  PortEntity     (集中端口)
  LineEntity     (分布参数线)
```

---

### 3.2 FieldEntity（场实体）

表示一个**可与路/线交换能量的场区域或场模式**。

**物理定义**

* 电场 **E**
* 磁场 **H**
* Poynting 通量

**接口量**

* 等效电压 V_f
* 等效电流 I_f
* 功率 P_f

**典型来源**

* FDTD：Huygens 面、探针面、等效端口
* MoM：基函数投影的等效场
* PEEC：partial inductance / capacitance 区域

---

### 3.3 PortEntity（端口实体）

表示一个**集中参数端口或网络节点**。

**物理定义**

* 端口电压 V
* 端口电流 I
* 参考方向

**接口量**

* (V, I)
* 等效阻抗 / 导纳

**典型来源**

* 电路网络（SPICE）
* Lumped port
* Feed port

---

### 3.4 LineEntity（传输线实体）

表示一个**分布参数系统**。

**物理定义**

* 多导体电压向量 **V(z, t)**
* 多导体电流向量 **I(z, t)**

**接口量**

* 端点 V/I
* 特征阻抗矩阵 Z₀
* 延迟 τ

**典型来源**

* MTL 求解器
* PEEC 导线模型

---

## 4. 统一耦合算子（最关键部分）

### 4.1 耦合算子抽象

所有耦合都写成如下形式：

```
⎡ q_field ⎤   ⎡  0     C_fc  C_fl ⎤ ⎡ x_field ⎤
⎢ q_port  ⎥ = ⎢ C_cf    0    C_pl ⎥ ⎢ x_port  ⎥
⎣ q_line  ⎦   ⎣ C_lf  C_lp    0   ⎦ ⎣ x_line  ⎦
```

其中：

* x：状态量（E/H，V/I）
* q：等效激励 / 反馈
* C_*：**耦合算子**

---

### 4.2 三类基本耦合算子

#### A. Field ↔ Port（场–路）

* 天线馈电
* 场激励电路
* 电路反射回场

**物理含义**

* 等效电压 = 场线积分
* 等效电流 = 磁通量

---

#### B. Field ↔ Line（场–线）

* 电缆辐射
* EMP 注入
* 外场耦合到线缆

**物理含义**

* 场 → 线：分布激励
* 线 → 场：等效电流源

---

#### C. Port ↔ Line（路–线）

* 网络端口接入传输线
* 端口反射、负载

---

## 5. 时间与频域的一致性规则

### 5.1 时间域（FDTD / 时域 MoM）

* 耦合量必须是：

  * V(t)
  * I(t)
* 不允许隐式未来量

### 5.2 频域（MoM / PEEC）

* 耦合量是：

  * V(ω)
  * I(ω)
* 时域 ↔ 频域通过 **显式变换模块**（L5）

🚫 禁止在接口层偷偷 FFT / IFFT

---

## 6. 接口 API 规范（示意）

### 6.1 Coupling Descriptor（声明式）

```yaml
coupling:
  type: field-port
  field_entity: fdtd_huygens_1
  port_entity: circuit_port_3
  direction: bidirectional
  reference:
    orientation: +z
    sign_convention: passive
```

---

### 6.2 Runtime Interface（抽象）

```c
typedef struct {
    double V;
    double I;
    double P;
} CouplingSignal;

void coupling_pull(FieldEntity*, CouplingSignal*);
void coupling_push(PortEntity*, const CouplingSignal*);
```

> 注意：
> **这里没有网格索引、没有矩阵、没有 GPU**

---

## 7. 在 FDTD / MoM 中的落地方式（不破坏架构）

### FDTD

* FieldEntity 由：

  * Huygens surface
  * Probe surface
* 转换发生在 **Operator 层**

### MoM / PEEC

* FieldEntity / LineEntity 由：

  * 基函数投影
  * Partial 元件
* 不直接暴露矩阵条目

### MTL

* LineEntity 只在端点暴露 V/I
* 内部传播由 MTL solver 自管

---

## 8. 能量一致性与验证规则

每一个耦合必须满足：

* 输入功率 = 输出功率 ± 数值误差
* 可做独立能量平衡测试
* 可关闭单个耦合进行对比

---

## 9. 明确禁止的耦合方式（高危）

❌ 在 FDTD kernel 中直接解电路
❌ 在 MoM matrix 中直接嵌入 Yee 更新
❌ 在 GPU kernel 中做场–路变换
❌ 在接口中传递网格索引、矩阵行号

---

## 10. 结论（设计核心）

> **统一耦合的本质不是“共享数据结构”，
> 而是：
> 用最小、最干净、最物理的接口，
> 连接彼此完全不同的数值世界。**

只要这个接口守住：

* FDTD 可以无限优化
* MoM 可以无限扩展
* Hybrid 可以长期演进而不崩

## 1️⃣ MoM Solver Pipeline（场积分 → 矩阵）

```
MoM Solver
│
├─ Geometry Interface
│
├─ Surface Meshing
│
├─ Basis Function (RWG / HO)
│
├─ Green's Function Evaluation
│
├─ Impedance Matrix Assembly (Z)
│
├─ Excitation Vector Assembly
│
├─ Linear Solver (GMRES / LU / FMM / ACA)
│
└─ Postprocessing
   ├─ Surface Current
   ├─ Near Field
   └─ Far Field / RCS
```

**工程属性**

* 频域为主
* 强依赖积分核与 Green 函数
* **不应感知 PEEC / FDTD / Ray 的存在**

---

## 2️⃣ PEEC Solver Pipeline（几何 → 等效电路）

```
PEEC Solver
│
├─ Geometry Interface
│
├─ Volume / Segment Discretization
│
├─ Partial Element Extraction
│   ├─ R
│   ├─ L / M
│   ├─ C
│   └─ G
│
├─ Circuit Topology Construction (MNA)
│
├─ Time / Frequency Solver
│
└─ Postprocessing
   ├─ Node Voltages
   ├─ Branch Currents
   └─ Power / EMC Metrics
```

**工程属性**

* 电路语义明确
* 允许非线性、时域
* **和 MoM 只在“耦合接口”层交互**

---

## 3️⃣ FDTD Solver Pipeline（网格 → 时域推进）

```
FDTD Solver
│
├─ Geometry Interface
│
├─ Uniform / Non-uniform Grid Generation
│
├─ Material Assignment
│
├─ Source Injection (Plane / Lumped / Cable)
│
├─ Time Marching (Yee / FVTD)
│
├─ Boundary Condition (PML / CPML)
│
└─ Postprocessing
   ├─ Time-domain Fields
   ├─ FFT / Wideband
   └─ Coupling Extraction
```

**工程属性**

* 时域为核心
* 适合 EMP / 雷击 / 强瞬态
* **可与 PEEC / Ray 做区域耦合**

---

## 4️⃣ Ray Solver Pipeline（几何光学 / 高频）

```
Ray Solver
│
├─ Scene / CAD / Terrain Input
│
├─ Region Classification
│
├─ Ray Launch
│
├─ Ray Tracing
│   ├─ Reflection
│   ├─ Diffraction
│   ├─ Transmission
│
├─ Path Accumulation
│
└─ Postprocessing
   ├─ Coverage / Field Strength
   ├─ Delay / Path Loss
   └─ Equivalent Sources
```

**工程属性**

* 高频、远场、统计性
* 可作为：

  * FDTD 的等效源
  * MoM 的远区激励
* **必须是独立 solver**

```

   ├─ --solver mom
   ├─ --solver peec
   ├─ --solver fdtd
```

* solver 间互相调用内部函数
