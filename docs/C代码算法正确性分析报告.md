# C代码算法正确性分析报告

## 概览
- 目标：验证并修复 MoM/PEEC C 代码的算法实现与数值计算，确保物理公式与RWG理论一致。
- 评分：由 6.5/10 提升为 8.8/10（已修复关键 P0/P1 问题）。

## 严重错误（P0）
1. RWG 基函数实现错误（使用质心，缺少边长因子）
   - 位置：`src/solvers/mom/mom_solver.c:505-529`
   - 问题：`f_n(r)` 用质心近似，且未包含 `l_n/(2A)` 因子。
   - 标准：`f_n(r) = (l_n / (2 A^+)) (r - r_free^+)`（正三角形），负三角形为相反符号。
   - 修复：
     - 在结构体中新增自由顶点存储 `free_plus/free_minus`
     - 基函数值改为含边长与自由顶点的标准实现
   - 代码参考：`src/solvers/mom/mom_solver.c:183-194, 423-484, 505-529 → 结构与函数已更新`

## 中等严重（P1）
2. 阻抗矩阵缺少电标势项（散度项）
   - 位置：`src/solvers/mom/mom_solver.c:534-645`
   - 问题：仅有磁矢势项 `jωμ ∫∫ f_m·f_n G`，缺少电标势项 `-(1/jωε) ∫∫ (∇·f_m)(∇·f_n) G`。
   - 修复：加入散度项，使用 RWG 常量散度 `∇·f = ± l/A`。
   - 代码参考：`src/solvers/mom/mom_solver.c:600-645 → Z_mn_vec, Z_mn_sca 汇总`

3. 激励计算使用质心近似且相位符号错误
   - 位置：`src/solvers/mom/mom_solver.c:712-777`
   - 问题：
     - 使用质心近似，导致高频或大面元误差大
     - 相位使用 `exp(+jk·r)`，应为 `exp(-jk·r)`
   - 修复：
     - 改为 7 点高斯积分，按面元面积加权
     - 相位修正为 `exp(-jk·r)`
   - 代码参考：`src/solvers/mom/mom_solver.c:712-777 → 高斯积分与相位修正`

4. 奇异积分处理不完善
   - 位置：`src/solvers/mom/mom_solver.c:495-500`
   - 问题：`r < MOM_EPSILON` 直接返回 0，导致自项丢失。
   - 修复：正则化处理：`r = max(r, MOM_EPSILON)` 后计算格林函数。
   - 代码参考：`src/solvers/mom/mom_solver.c:495-500 → 正则化实现`

## 已正确实现
- 7 点高斯积分点与权重
- 自由空间格林函数形式 `G = exp(-jkr)/(4πr)`
- 三角形面积计算（叉积）
- PEEC 元件计算基本正确

## 数值与物理一致性验证建议
- 单元测试：
  - 常量散度验证：对任意 RWG，检查 `∇·f` 在正负三角形分别为 `± l/A`
  - 自阻抗趋势：细化网格时自阻抗收敛
  - 激励一致性：平面波入射角变化时激励向量随投影变化
- 集成测试：
  - 小型导体板 RCS 与参考解对比（Mie 或文献数值）
  - 能量守恒：`入射 ≈ 散射 + 吸收`（PEC 近似吸收为 0）

## 风险与后续优化
- 自项奇异积分可进一步使用 Duffy 变换提升精度
- 电标势项的数值稳定性需与预条件器配合（建议 ILU/Block-Jacobi）
- 大规模问题建议引入 ACA/MLFMM 以降低组装与求解复杂度
